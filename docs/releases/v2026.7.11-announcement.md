# v2026.7.11 Release Announcement

**Release Date:** 2026-07-21

Fortemi v2026.7.11 is the stable publication recovery for the signed
`v2026.7.10` data-portability release. It carries the same validated Knowledge
Shard contracts and nine-cell producer/consumer conformance matrix while
repairing the final registry and release-asset publication paths.

## Highlights

| Change | Why You Care |
|--------|--------------|
| **Complete registry publication** | The resolved release version is now exported to the GHCR promoter, enabling versioned API and multi-architecture bundle images. |
| **Mirrored native evidence** | Gitea and GitHub releases carry the same three native binaries, checksum manifest, and in-toto provenance statement. |
| **Fail-closed asset parity** | Publication rejects checksum failures, provenance-subject mismatches, and differing pre-existing GitHub assets. |
| **Unchanged data contract** | `core-v1`, reduced `record-v1`, server `full-v1`, and the registered `1.0.0 -> 1.1.0 -> 1.2.0` migration window are unchanged from `2026.7.10`. |

## Compatibility Boundary

- `core-v1` remains the default structured-record and attachment-projection
  profile.
- `record-v1` remains the deliberate RecordStore subset with mandatory loss
  reporting.
- `full-v1` remains supported for the complete Fortemi server self-route;
  external products do not advertise that profile.
- The public Fortemi React `2026.7.11` package remains the locked browser
  consumer boundary used by the cross-layer release evidence.

## Upgrade Notes

Use `2026.7.11` for stable deployments. The signed `v2026.7.10` tag remains an
immutable source receipt, but its tag workflow did not complete GHCR
publication. No shard conversion or schema migration beyond the existing
registered chain is required.

```bash
docker pull ghcr.io/fortemi/fortemi:2026.7.11
docker pull ghcr.io/fortemi/fortemi:bundle-2026.7.11
```
