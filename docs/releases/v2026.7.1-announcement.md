# v2026.7.1 Release Announcement

**Release Date:** 2026-07-13

Fortémi v2026.7.1 strengthens in-place upgrades from February releases and
adds a supported Intel Arc/XPU deployment path using host vLLM. It also moves
Gitea release automation to OpenBao-backed credentials and defines the portable
binary-attachment Knowledge Shard contract.

## Highlights

| Change | Why You Care |
|--------|--------------|
| **February upgrade hardening** | Existing bundle volumes repair legacy role authentication before backup and migration; constrained backup staging now degrades explicitly and safely. |
| **Complete bulk reprocessing** | Archive-wide reprocessing paginates beyond the former 100-note ceiling. |
| **Intel Arc / XPU deployment** | A validated Compose overlay and corrected sync/stream chat routing send generation to host vLLM without requesting NVIDIA devices, with a systemd template and smoke test. |
| **OpenBao-backed release CI** | Build, publication, documentation, and release jobs obtain scoped credentials at runtime. |
| **Portable byte-sidecar format** | A self-contained shard may store each distinct attachment blob once at `blobs/<hash>`, without putting bytes in JSON records. |
| **Canonical BLAKE3 encoding** | Attachment records use `blake3:<64-char-lowercase-hex>`; sidecar entries use the same digest without the prefix. |
| **Display paths only** | Attachment projection paths identify the displayed filename, never a filesystem path, OPFS key, or object-store key. |
| **Reference-only shards remain valid** | Missing sidecar entries preserve today's behavior and do not make a shard malformed. |
| **Truthful server boundary** | The server currently exports reference-only shards and does not restore attachment records or bytes during shard import. |

## Upgrade Notes

Bundled deployments upgrading an existing database should retain the documented
pre-migration acknowledgments and provide enough space for the verified backup.
The bundle now tries compressed RAM-backed staging first and emits a prominent
warning before falling back to temporary disk staging under `BACKUP_DEST`.

Intel Arc/XPU deployments require Docker Compose v2.17.0 or newer. Start Fortémi
with `docker-compose.bundle.yml` plus `docker-compose.intel.yml`, run vLLM on the
host, and keep embeddings on Ollama or another embedding-capable provider. See
`docs/content/intel-arc-vllm.md` for the complete configuration and validation
steps.

## Portable Attachment Contract Boundary

This release ratifies the portable format; it does not add server-side
self-contained shard export or attachment-byte restoration. That follow-up
requires bounded streaming, digest verification, deduplication, attachment
restoration, and archive round-trip tests before the server can claim byte
portability.

Existing reference-only shards remain valid. Consumers implementing the
optional sidecar should follow `docs/content/binary-attachment-projection.md`
and continue to reject raw byte fields in JSON projection records.

To pull the versioned images after publication:

```bash
docker pull ghcr.io/fortemi/fortemi:2026.7.1
docker pull ghcr.io/fortemi/fortemi:bundle-2026.7.1
```
