# v2026.7.0 Release Announcement

**Release Date:** 2026-07-12

Fortémi v2026.7.0 is a migration-safety and release-publishing recovery release.
It restores the upgrade path from the February `v2026.2.0` database state to
current, adds a fail-closed pre-migration backup gate for bundled deployments,
and ships the audited migration evidence needed before refreshing public GHCR
release images.

## Highlights

| Change | Why You Care |
|--------|--------------|
| **February-to-current upgrade proven** | Exact `v2026.2.0` migration history can now reach current without manual database surgery. |
| **Pre-migration backup gate** | Bundled deployments must create a backup before migrations run; failed backups stop startup instead of risking an unsafe upgrade. |
| **Legacy SKOS trigger repair** | Restores from old pgvector trigger definitions are normalized before backup/migration. |
| **Release publishing recovery** | Versioned GHCR release images are cut after the migration audit and dev publish verification, avoiding premature public refresh. |

## Upgrade Notes

Bundled deployments upgrading from February-era releases should follow
`docs/ops/feb-to-current-upgrade-runbook.md`. The bundle entrypoint now creates a
pre-migration backup before applying database migrations and exits non-zero if
that backup cannot be written.

For release verification, record immutable image digests after pulling:

```bash
VERSION=2026.7.0
docker pull ghcr.io/fortemi/fortemi:${VERSION}
docker pull ghcr.io/fortemi/fortemi:bundle-${VERSION}
docker image inspect ghcr.io/fortemi/fortemi:${VERSION} --format '{{index .RepoDigests 0}}'
docker image inspect ghcr.io/fortemi/fortemi:bundle-${VERSION} --format '{{index .RepoDigests 0}}'
```
