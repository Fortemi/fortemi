## Strict Tag Filtering

The headline feature of this release enables guaranteed data isolation using SKOS concepts and schemes.

### Use Cases

- **Multi-tenancy**: Isolate client data without separate databases
- **Project segregation**: Keep project notes strictly separated
- **Compliance**: Meet GDPR/SOC2 data access requirements

### How It Works

Strict filters apply a WHERE clause **before** fuzzy search, guaranteeing results match criteria exactly:

```json
{
  "query": "authentication",
  "strict_filter": {
    "required_tags": ["project:matric"],
    "any_tags": ["priority:high", "priority:critical"],
    "excluded_tags": ["status:archived"],
    "required_schemes": ["client-acme"]
  }
}
```

### Filter Types

| Filter | Logic | Description |
|--------|-------|-------------|
| `required_tags` | AND | Notes MUST have ALL these |
| `any_tags` | OR | Notes MUST have AT LEAST ONE |
| `excluded_tags` | NOT | Notes MUST NOT have ANY |
| `required_schemes` | Isolation | Notes ONLY from these vocabularies |
| `excluded_schemes` | Exclusion | Notes NOT from these vocabularies |

### MCP Tools

Two new tools expose strict filtering:

```javascript
// Option 1: Add to existing search
search_notes({
  query: "API design",
  strict_filter: { required_tags: ["project:matric"] }
})

// Option 2: Dedicated strict search tool
search_notes_strict({
  query: "API design",
  required_tags: ["project:matric"],
  excluded_tags: ["draft"]
})
```

## SKOS Tagging System

Full W3C SKOS-compliant hierarchical tagging:

- **Concepts** with broader/narrower/related relations
- **Concept Schemes** for vocabulary organization
- **Governance** with candidate/controlled/deprecated status
- **PMEST Facets** for faceted classification

## Hybrid Search

Three search modes optimized for different use cases:

| Mode | Best For |
|------|----------|
| `hybrid` (default) | General search - combines keyword + semantic |
| `fts` | Exact keyword matching |
| `semantic` | Conceptually related content |

## MCP Server

65+ tools organized into categories:

- **Notes** (12): CRUD, search, export
- **Collections** (6): Hierarchical folders
- **Templates** (5): Reusable structures
- **SKOS Concepts** (20): Tag management
- **Backup/Export** (15): Data portability
- **Jobs** (3): Background processing
- **Encryption** (6): PKE operations

## PKE Encryption

Secure note sharing with public-key encryption:

- X25519 ECDH key exchange
- AES-256-GCM symmetric encryption
- Multi-recipient envelope encryption
- Wallet-style addresses with checksums

## Inference Backends

Pluggable LLM backends:

- **Ollama** (default) - Local inference
- **OpenAI-compatible** - OpenAI, vLLM, OpenRouter, LocalAI

## Installation

### API Server

```bash
git clone https://github.com/fortemi/fortemi
cd matric-memory
cargo build --release

# Set environment
export DATABASE_URL="postgres://user:pass@localhost/matric"

# Run
./target/release/matric-api
```

### MCP Server

Add to Claude Desktop config:

```json
{
  "mcpServers": {
    "matric-memory": {
      "command": "node",
      "args": ["/path/to/matric-memory/mcp-server/index.js"],
      "env": {
        "MATRIC_MEMORY_URL": "http://localhost:3000"
      }
    }
  }
}
```
