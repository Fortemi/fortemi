# v2026.7.10 Release Announcement

**Release Date:** 2026-07-21

Fortemi v2026.7.10 establishes reproducible data portability across the
registered server, React, PGlite, RecordStore, and AIWG Knowledge Shard layers.
Every required matrix cell is backed by digest-pinned archives and receipts,
with clean-destination import, semantic re-export, and zero-mutation rejection
assertions enforced before compatibility, portability, backup, or parity claims
are allowed.

## Compatibility Boundary

- `core-v1` is the default export and carries structured records plus attachment
  projections. Schema `1.0.0`, `1.1.0`, and `1.2.0` inputs are accepted only
  through the exact registered migration chain.
- `record-v1` is the deliberate RecordStore subset. Its producer must report
  every omitted richer concept, and schema `1.0.0` is rejected because that
  profile was not defined there.
- `full-v1` is supported for the Fortemi server self-route and requires every
  rich component and declared attachment byte. External products do not yet
  advertise `full-v1`; their producer and consumer receipts remain pending.
- A default `core-v1` export is not a disaster-recovery-complete archive. Select
  `profile=full-v1` explicitly when the complete server profile is required.

## Upgrade Notes

No manual shard conversion is required. Import validates the original archive,
applies only registered migrations, revalidates the migrated representation,
and performs persistent writes atomically. Retain original archives as immutable
receipts and test restores against the profile your deployment depends on.

To pull the versioned images after publication:

```bash
docker pull ghcr.io/fortemi/fortemi:2026.7.10
docker pull ghcr.io/fortemi/fortemi:bundle-2026.7.10
```
