## Complete Media Pipeline

The headline theme of this release. Fortemi now handles the full lifecycle of media content — from resumable upload through transcription, diarization, keyframe analysis, variant generation, and streaming playback.

### TUS Resumable Uploads (#528)

Standards-compliant [tus v1.0.0](https://tus.io/) resumable file upload protocol with three extensions:

- **Creation** — `POST /api/v1/attachments/{note_id}/tus` returns an upload URL
- **Termination** — `DELETE /tus/{id}` cancels an in-progress upload
- **Checksum** — Optional client-side integrity verification

Upload sessions survive network interruptions. Clients resume from the last successful byte with a `HEAD` request to check the current offset. Sessions expire after 24 hours by default.

### Speaker Diarization Pipeline (#497)

pyannote-based speaker identification runs as a GPU sidecar container. After Whisper transcription, the diarization pipeline identifies and labels individual speakers:

- Speaker-labeled VTT, SRT, and TXT caption files as derived attachments
- Speaker configuration block in note content for display
- Editable speaker names via `SpeakerRelabel` job
- Configurable via `DIARIZATION_BASE_URL` (defaults to `http://pyannote:8001`)

### Media Optimize Handler (#506)

Pre-generates streaming-friendly media variants during attachment upload using ffmpeg:

| Variant | Description |
|---------|-------------|
| `faststart` | MP4 moov atom relocation for progressive download |
| `web_compatible` | MKV/MOV remuxed to H.264+AAC MP4 |
| `preview_720p` | Downscaled 720p H.264 preview |
| `audio_only` | Audio track extracted from video |
| `web_audio` | AAC transcode for universal playback |
| `audio_preview` | Lower-bitrate audio preview |

Variants stored as derived attachments and served via `?variant=` query parameter on the download endpoint. Defaults to enabled for video/audio uploads (`media_optimize=true`).

### Atomic Keyframe Vision Pipeline (#526, #529)

Keyframe vision descriptions now process as individual `KeyframeVision` jobs instead of inline during extraction:

- Per-frame checkpointing — individual frame failures don't block the pipeline
- Parallel processing across available GPU resources
- Fan-in assembly — `KeyframeAssembly` triggers automatically when all frames are described
- Always registered regardless of vision backend availability (#529) — jobs defer with `Retry` until the backend is configured

### Thumbnail Sprite Sheets (#525)

CSS sprite sheets generated from video keyframes with WebVTT timestamp maps:

- 5x5 JPEG grids (25 thumbnails per sheet, 160x90 each)
- WebVTT map at `/api/v1/attachments/{id}/thumbnails.vtt`
- Sprite images at `/api/v1/attachments/{id}/sprites/{index}`
- Enables seek-bar thumbnail previews in UI clients

## Document Extraction

### Email Extraction (#508-#512)

RFC 2822 / MIME email parsing for `.eml` and `.mbox` files. Extracts message body, headers, and binary attachments as derived child attachments that trigger their own extraction jobs.

### Spreadsheet Extraction (#508-#512)

Excel (`.xlsx`, `.xls`) and ODS spreadsheet extraction. Converts each sheet to markdown tables.

### Archive Extraction (#514-#515)

ZIP, tar, and gzip archive extraction. Produces file listing with text content extraction (capped at 1000 files, 100 MB total).

## Framework Upgrade

### axum 0.7 to 0.8 (#524)

Major dependency upgrade across the HTTP stack:

| Package | Before | After |
|---------|--------|-------|
| axum | 0.7 | 0.8 |
| tower | 0.4 | 0.5 |
| tower-http | 0.5 | 0.6 |
| tokio-tungstenite | 0.24 | 0.28 |

All `Service` implementations adapted to the `call(&self)` signature change (previously `call(&mut self)`).

## Upgrade Notes

1. **Database migrations run automatically** on startup via `sqlx::migrate!()`
2. **pyannote sidecar** is new in this release. Set `DIARIZATION_BASE_URL=` (empty) to disable speaker diarization.
3. **TUS uploads** available at `/api/v1/attachments/{note_id}/tus`. Existing multipart upload unchanged.
4. **Media optimization** enabled by default for video/audio uploads. Set `media_optimize=false` on upload to skip.
5. **axum 0.8 upgrade** — No API-breaking changes for consumers. Internal `Service` trait change only.
6. **KeyframeVision** (#529) — If you previously had `OLLAMA_VISION_MODEL` unset, keyframe description jobs were silently orphaned. They will now auto-retry when the vision backend is configured.
7. **MCP core tools** expanded to 38 (added `bulk_reprocess_notes`).
