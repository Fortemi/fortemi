# Documentation Sync Report - 2026-07-13

## Scope

- Audited code: shard export/import paths, attachment projection construction,
  BLAKE3 content hashing, February-baseline bundle upgrade and backup paths,
  bulk reprocessing, inference provider selection, Intel Arc/XPU deployment
  assets, and release automation
- Audited docs: binary attachment contract and evidence, backup guide,
  February upgrade evidence, Intel deployment guide, documentation manifest,
  release guide, and release-facing changelog/announcement lane

## Findings And Fixes

1. The optional portable shard sidecar was a canonical format contract, but the
   server currently emits reference-only shards and does not restore attachment
   records or bytes. The canonical page, architecture artifact, test plan,
   evidence samples, backup guide, and manifest now state that boundary.
2. The flow text conflated byte-free JSON projection builders with a future
   sidecar archive writer. The two responsibilities are now separated.
3. Earlier projection tests do not prove sidecar round-tripping. The test plan
   now identifies the missing exporter/importer proof without reopening the
   accepted projection contract.
4. Two credential-shaped fixture DSNs failed the blocking documentation
   contract. The published command now uses `<DATABASE_URL>` and the executable
   fixture assembles its test DSN without publishing a credential-shaped
   literal; behavior is unchanged.
5. The February-baseline role-authentication, backup-staging, and bulk
   reprocessing fixes are represented in the smoke-test report and release
   artifacts. No additional operator-contract correction was required.
6. The Intel Compose overlay is represented in the environment example,
   hardware and inference guides, dedicated deployment guide, release
   checklist, manifest, validator, and smoke test. No missing public setup or
   verification step was found.
7. Release validation found that runtime provider selection and effective
   config reporting contradicted the documented `MATRIC_INFERENCE_DEFAULT`
   contract. The provider registry and config response now honor configured
   env-only OpenAI defaults, with fallback and smoke-test coverage.
8. A production-layout candidate build found that repeated bare Docker build
   arguments erased optional-component defaults under the legacy builder. The
   redundant declarations were removed so fresh bundle databases retain
   PostGIS and can complete migrations without explicit CI-only overrides.
9. The first Intel traffic smoke proved that chat handlers bypassed the selected
   provider registry and constructed Ollama directly. Synchronous and streaming
   chat, health reporting, and model metadata now use the common generation
   backend; OpenAI-compatible backends retain native token streaming.
10. The candidate bundle build exposed a missing `.dockerignore` that sent more
    than 9 GB of local Rust artifacts to Docker. Build outputs, dependency
    directories, VCS metadata, logs, and local secrets are now excluded from
    image contexts.
11. The production bundle compiled `matric-inference` with its Ollama-only
    default feature set, so the configured OpenAI provider was visible but
    could not be constructed. The API dependency now enables the OpenAI backend
    required by the shipped Intel host-vLLM overlay.
