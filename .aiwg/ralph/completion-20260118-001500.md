### Phase 5: Knowledge Health Dashboard (#103)
- Overall health score (0-100) based on staleness, linkage, and tagging
- `GET /api/v1/health/knowledge` - summary metrics
- `GET /api/v1/health/orphan-tags` - tags used only once
- `GET /api/v1/health/stale-notes` - notes not updated in 90+ days
- `GET /api/v1/health/unlinked-notes` - notes with no links
- `GET /api/v1/health/tag-cooccurrence` - tag pair frequency
- Modified: `crates/matric-api/src/main.rs`

### Phase 6: Rate Limiting (#60)
- Governor-based token bucket rate limiting
- Configurable via env vars:
  - `RATE_LIMIT_REQUESTS`: requests per period (default: 100)
  - `RATE_LIMIT_PERIOD_SECS`: period in seconds (default: 60)
  - `RATE_LIMIT_ENABLED`: toggle (default: true)
- Returns HTTP 429 when rate exceeded
- Status endpoint: `GET /api/v1/rate-limit/status`
- Modified: `crates/matric-api/src/main.rs`, `Cargo.toml`
